# `build.mps` — Source of Truth for the Ant Pipeline

All `build/mps*.xml` Ant scripts carry a `<!-- Generated by MPS -->` header. The single source of truth is the MPS build-language model:

- `plugins/mps-build/solutions/mpsBuild/models/build.mps`

It is authored in `jetbrains.mps.build` + `jetbrains.mps.build.mps` + `jetbrains.mps.build.workflow` + `jetbrains.mps.core.xml` + `jetbrains.mps.build.startup`.

The model contains many `BuildProject` nodes (concept `jetbrains.mps.build.structure.BuildProject`, index `1l3spW`). Each `BuildProject` has three key properties:

- `TrG5h` — its symbolic name (e.g. `mpsHttpSupportPlugin`)
- `turDy` — the **target Ant file name** (e.g. `mpsHttpSupportPlugin.xml`)
- `2DA0ip` = `../../../../` (path relative to the model — emits the Ant file under `<MPS>/build/`)

Regenerating the model writes/refreshes the corresponding `build/*.xml` Ant scripts.

> **Do not hand-edit the generated Ant files** as a permanent solution — author the change in `build.mps` and regenerate.

## Note on opaque concept indices

Opaque codes like `1l3spW`, `398b33`, `2sgV4H`, `TrG5h`, `turDy`, `2DA0ip`, `1l3spV`, `10PD9b` are MPS concept / property / link ids. They are stable identifiers in MPS metadata; the qualified name is shown alongside in each reference. You only need them when reading the raw `.mps` XML or writing generator templates against the build language — the MPS editor displays qualified names.
